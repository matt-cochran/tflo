# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in tflo, please report it by emailing
**security@tflo.dev**.

We ask that you **do not** disclose the vulnerability publicly until we have had
a chance to address it. We practice **coordinated disclosure** — we will work
with you to understand the issue, develop a fix, and coordinate the public
announcement.

### What to include in your report

- A clear description of the vulnerability
- Steps to reproduce the issue
- The version(s) affected
- Any potential impact you have identified
- Your name and contact details (optional, if you'd like credit)

## Response Timeline

You can expect an initial response within **48 hours** of submitting your
report. We will keep you informed of our progress as we work toward a fix.

Our typical response cadence:

| Timeframe | Action |
|-----------|--------|
| < 48 hours | Initial acknowledgment and triage |
| 1–7 days | Investigation and reproduction |
| 7–21 days | Development of a fix and release planning |
| Variable | Coordinated public disclosure |

## Supported Versions

Only the **latest stable release** of tflo is supported with security updates.

| Version | Supported |
|---------|-----------|
| Latest stable release | ✅ |
| Older releases | ❌ |

We encourage all users to stay up-to-date with the latest release to ensure they
receive any security patches.

## Disclosure Policy

We follow a **coordinated disclosure** process:

1. The reporter submits a vulnerability report.
2. We acknowledge receipt and begin investigating.
3. We develop a fix and prepare a patched release.
4. We coordinate with the reporter on the release timeline.
5. Once the fix is released, we publish a security advisory and the reporter
   may publicly disclose the issue.

We will make every effort to resolve the issue quickly and to give the reporter
appropriate credit for their discovery.
