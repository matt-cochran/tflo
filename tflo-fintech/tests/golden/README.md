# Golden Tests

Golden tests validate `tflo-core` indicator behavior against reference output from the TA-Lib C library. The fixture JSON files are generated by `scripts/generate_talib_vectors.py`; Rust tests run real `tflo-core` graph computations through `runner.rs` and compare against those fixtures.

## Single-Input Fixture Contract

The fixture schema intentionally stores one primary `input` series. Indicators that require OHLCV data derive synthetic fields identically in Python and Rust:

- `close = input`
- `high = close * 1.01`
- `low = close * 0.99`
- `volume = 1.0`

Keep this rule synchronized between:

- `scripts/generate_talib_vectors.py`
- `runner.rs::build_records`

## TA-Lib Compatibility Notes

These are the edge cases we have already tripped over. Use these rules when adding or debugging golden vectors.

### OBV

TA-Lib `OBV` does **not** start at zero. It initializes the first output to the first bar's volume, then applies the usual cumulative sign logic for later bars:

- first bar: `obv = volume[0]`
- up bar: `obv += volume[i]`
- down bar: `obv -= volume[i]`
- unchanged bar: `obv` unchanged

With the golden-test constant-volume rule (`volume = 1.0`), the first expected value is therefore `1.0`, not `0.0`. The `Comp::obv` implementation uses a stateful scan to match this TA-Lib baseline exactly.

### MFI

TA-Lib `MFI` is **not** a Wilder-smoothed RSI of raw money flow. It uses rolling sums of positive and negative raw money flow over `period` typical-price changes:

1. `typical_price = (high + low + close) / 3`
2. `raw_money_flow = typical_price * volume`
3. If `typical_price[i] > typical_price[i - 1]`, raw flow contributes to the positive sum.
4. If `typical_price[i] < typical_price[i - 1]`, raw flow contributes to the negative sum.
5. `MFI = 100 - 100 / (1 + positive_sum / negative_sum)`

For a `period = 14`, the first TA-Lib output appears at index `14`, after 14 close-to-close typical-price changes. The `Comp::mfi_n` implementation uses a stateful scan to match this rolling-sum behavior.

### CMO

TA-Lib `CMO` is not the simple rolling-sum version sometimes shown in indicator references. It is RSI-like:

1. Seed average gain/loss with simple averages over the first `period` close-to-close differences.
2. Emit the first value at index `period`.
3. Update subsequent average gain/loss values with Wilder smoothing.
4. Compute `CMO = 100 * (avg_gain - avg_loss) / (avg_gain + avg_loss)`.

This makes TA-Lib CMO effectively a centered Wilder RSI transform, not a pure rolling window sum. `Comp::cmo_n` intentionally follows TA-Lib for golden parity.

## Running

From the workspace root:

```bash
cargo test -p tflo-core --test golden -- --nocapture
```

In environments where the tool runner cannot enter the workspace root directly, use the manifest path:

```bash
cargo test --manifest-path /home/mc/working/tflo/Cargo.toml -p tflo-core --test golden -- --nocapture
```

## Regenerating Fixtures

Requires TA-Lib C library and Python dependencies. From anywhere:

```bash
python3 /home/mc/working/tflo/tflo-core/tests/golden/scripts/generate_talib_vectors.py --all --output-dir /home/mc/working/tflo/tflo-core/tests/golden/fixtures
```

Do not generate expected vectors from `tflo-core` output; TA-Lib is the reference source.
